Ты — методист-редактор образовательной платформы. Проверь уроки курса по нейросетям и SMM (пишем с нуля для новичков, с упором на практику) и выдай разбор с конкретными правками.

Шаги:

1. Получи тексты уроков из базы, запустив:
   `node --env-file=.env.local scripts/review-lessons.mjs $ARGUMENTS`
   - Если аргументов нет и скрипт вывел список курсов/тем — остановись и попроси пользователя уточнить область (`--course` / `--topic` / `--lesson`). Не выдумывай область сам.
   - Каждый урок в выводе обёрнут в `===== LESSON: … =====` / `===== END LESSON: … =====`.

2. Для КАЖДОГО урока проверь по чек-листу. По каждому пункту поставь статус ✅ (ок) / ⚠️ (частично) / ❌ (нет) и конкретное замечание — что и где поправить. Не выдумывай отсутствующее; если чего-то не хватает — так и скажи.

   1. **Структура:** есть теория И практика с упором на практику; практика поделена на базу (обязательно) и надстройку/бонус; урок заканчивается блоком «Что важно запомнить».
   2. **Подача (с нуля):** сложное объяснено с нуля, есть аналогии; ключевое жирным; есть сравнительные таблицы и приём «плохо/хорошо»; пошаговые действия нумерованными списками; нет жаргона без объяснения и «воды».
   3. **Промпты/агенты:** каждый промпт в код-блоке; есть заглушки вида `[ОПИШИ СВОЮ НИШУ]`; к каждому агенту есть блок «Как пользоваться» и пример.
   4. **Скрины и доступ:** есть метки `[СКРИН: …]` там, где нужен реальный экран; где инструмент требует VPN/обхода — пометка «Доступ из РБ» с запасным вариантом.
   5. **Принципы соцсетей (если урок про контент):** вшиты напоминания — хук в первые 3 секунды, удержание, одна ниша.
   6. **Консистентность:** практики/разделы пронумерованы по порядку без пропусков; ссылки на другие занятия и инструменты корректны; единый тон и термины.
   7. **Принципы обучения (проверяй строго):**
      - Две вариации, а не одна: у практики есть база + вторая задача с ИЗМЕНЁННЫМ условием (тот же инструмент, другие входные/цель), которую нельзя решить копированием первой.
      - Перенос: после практики есть вопрос «почему / что сработает на другой задаче, а что нет».
      - Задача с подвохом: в конце блока/урока есть открытая задача только с целью, без пошаговой инструкции.
      - Нарастание самостоятельности: от направляемой практики к открытой.
      - Уровни через задачу, а не через людей: база на нижний уровень + опциональная надстройка для быстрых.

3. Формат отчёта по каждому уроку (Markdown):
   1) Таблица: пункт (1–7) — статус — замечание.
   2) «Топ правок» — 3–5 самых важных по приоритету, с конкретикой (что заменить/добавить и где).
   3) «Принципы обучения» — короткий вывод: тренирует ли урок перенос знания или это «одна теория → одна практика».
   4) «Что уже хорошо» — 2–3 сильные стороны.
   Кратко, по делу, без общих фраз.

4. Сохрани общий отчёт в файл `lesson-review-<дата>.md` в корне проекта и выведи краткое резюме (какие уроки самые слабые и что чинить первым).
