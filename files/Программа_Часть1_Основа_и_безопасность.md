# Часть 1 — Основа и безопасность (2 часа)

> Опорная методичка: последовательность «что за чем». Содержание модулей —
> М0 (как говорить с ИИ + агент) и М1 (обезличивание перед облаком).
> Все опоры и промпты — здесь; макрос — файл `Анонимизация.bas`.

**Тайминг:** вступление и вход 15 · М0 База 45 · М1 Анонимизация 25 ·
агент «Генератор макросов» 20 · итог и мост 15.

---

## 0. Вступление и вход (15 мин)

**Последовательность:**
1. Цели дня и формат (3 части по 2 часа). Ключевая рамка: **ИИ = педантичный
   стажёр** — не врёт, всегда показывает, откуда взял; финальное решение за аудитором.
2. Правило безопасности одной фразой: **в облако — только обезличенные данные**
   (детально в М1).
3. Вход по шагам (проговорить и дождаться, пока у всех получилось):
   - включить **VPN**;
   - зайти в первый **аккаунт Google** (второй — про запас, для лимитов);
   - открыть **gemini.google.com** на десктопе;
   - показать, где **Gems Manager** (там будем создавать агентов).

**LMS-опора:** карточка «Вход» (VPN → аккаунт → Gemini → Gems Manager);
карта «какой инструмент под что».

## М0. База ИИ-грамотности (45 мин)

### Подцель
Промпт-инжиниринг, ментальная модель агента, понимание «где какой бесплатный
инструмент». **Переносимый навык** — работает в любых задачах, не только аудит.

### LMS-опора (лежит на платформе, открыта во время занятия)
- **Формула промпта Р·З·К·Ф:**
  - **Р — Роль:** кто ИИ в этой задаче («Ты — ассистент аудитора…»).
  - **З — Задача:** что конкретно сделать, одним ясным действием.
  - **К — Контекст:** входные данные, ограничения, правила («существенное — дословно»,
    «нет данных — пиши "не указано"»).
  - **Ф — Формат:** в каком виде ответ (таблица, строка через табуляцию, три блока).
- **Чек-лист «признаки рабочего промпта»:** задана роль · одна чёткая задача ·
  есть контекст и правила · задан формат вывода · сказано, что делать при нехватке данных.
- **Определение:** *агент = сохранённый системный промпт (Gem)*. Собрал один раз —
  пользуешься всегда.
- **Карта «какой инструмент под что»:** Gemini/Gems — агенты; NotebookLM — документы;
  Flash — модель по умолчанию, для наших задач хватает.

### Демо (последовательность шагов)
1. Взять их материал (обезличенный) и задать запрос **«в лоб»** («сделай таблицу по
   договору») → показать, что ответ рыхлый/неполный.
2. Тот же запрос **по формуле Р·З·К·Ф** → показать, насколько ответ собраннее.
   Акцент: разница даёт **контекст и формат**.
3. Механика **создания Gem:** Gems Manager → New Gem → вставить инструкцию → сохранить →
   показать, что теперь агент готов принимать данные.

### Практика (шаги)
1. Каждый пишет промпт по формуле на микро-задаче (например: «переписать заметку в
   деловое предложение из 3 пунктов»).
2. Разбор вживую одной-двух работ. **Типовая ошибка уровня «что-то трогали»:** нет
   контекста и формата → чиним, добавляя К и Ф, показываем, как меняется ответ.

### Переносимость (проговорить отдельно)
Эти приёмы работают в любых задачах — письма, планы, разбор документов, личные
вопросы. Это фундамент, к нему возвращаемся весь курс.

### Материалы модуля
Карточка Р·З·К·Ф · чек-лист «признаки рабочего промпта» · карта инструментов ·
инструкция «New Gem».

## М1. Анонимизация — фундамент безопасности (30 мин)

### Подцель
Обезличить данные перед заливкой в облако. Для аудитора — обязательный шаг,
без него ничего в нейросеть не уходит.

### LMS-опора
- **Правило:** реальные данные в облако не уходят — только токены (КОМПАНИЯ_001,
  СЧЁТ_003).
- **Чек-лист «что маскируем»:** наименование, УНП, р/с, IBAN, ФИО, БИК,
  чувствительные суммы.
- **Макрос:** `Анонимизация.bas` (две команды — `Обезличить` и `ВернутьДанные`).

### Демо (последовательность шагов)
1. `Обезличить` на реальной табличке → выделяешь диапазон → вводишь префикс токена →
   данные заменяются токенами, появляется лист **«Карта_замен»**.
2. Показать **«Карта_замен»** и подчеркнуть: этот лист в нейросеть **не** загружаем.
3. `ВернутьДанные` → на ответе с токенами → показать обратную подстановку оригиналов.

### Практика (шаги)
1. Каждый импортирует макрос (инструкция ниже) и гоняет `Обезличить` на учебном файле.
2. Возвращает данные через `ВернутьДанные`.
3. **Типовая ошибка:** потеряли/загрузили карту замен → проговорить, почему это провал
   безопасности.

### Запасной путь (если макросы заблокированы корп-политикой)
Ручной **«Найти и заменить»** по той же логике: составить список реальных значений →
заменить на токены → вести карту замен вручную в отдельном файле. Медленнее, но
безопасность та же.

### Честный нюанс (обязательно проговорить)
Макрос маскирует **значение ячейки целиком**. Если контрагент или УНП сидят
**внутри длинной строки** (содержание, субконто) — вкрапление он не поймает.
Свободный текст проверяем **глазами**.

### Инструкция: как завести любой наш макрос
1. Открыть нужный файл → `Alt+F11` (редактор VBA).
2. File → Import File → выбрать `.bas` (или вставить код в новый модуль: Insert → Module).
3. Закрыть редактор → `Alt+F8` → выбрать процедуру (`Обезличить` / `ВернутьДанные`) → Run.
4. Если Excel ругается на макросы — включить их для файла (Разработчик → Безопасность
   макросов) или идти запасным ручным путём.

### Материалы модуля
Правило «только токены» · чек-лист маскировки · `Анонимизация.bas` · учебный файл ·
инструкция импорта макроса.

## Агент «Генератор макросов» — продолжение М1 (20 мин)

### Подцель
Показать, **откуда берётся** макрос анонимизации, и дать переносимый навык: аудитор
описывает задачу словами → агент выдаёт готовый макрос + инструкцию вставки.
Промпт — `Промпты_агентов_Gem`, №11 (под Windows — им пользуются сами аудиторы).

> Формат — демо-ведомый, НЕ полная самостоятельная генерация у всех сразу.
> Для уровня «что-то трогали» массовая генерация макросов = хаос из ошибок.
> Каждый уносит агента на полку и генерит своё уже после, в спокойном темпе.

### Демо (последовательность шагов)
1. Связка с М1: «макрос анонимизации не с неба упал — его можно получить у агента».
2. Создать Gem «Генератор макросов» (промпт №11).
3. Подать конкретную задачу словами — по формуле входа (см. ниже) на общем экране.
   Пример: «на листе Данные, колонка D — суммы; выдели жёлтым строки, где сумма выше
   100000; исходные данные не трогай».
4. Показать ответ агента: «что сделает» → код → «как запустить» → «что поменять».
5. Подчеркнуть: агент **переспрашивает** про лист и колонки, если их не назвали —
   и это правильно, иначе макрос не заведётся (ровно та дисциплина из нашего теста).

### Формула входа (главное — этому и учим)
Хороший запрос агенту = 4 вещи: **(1)** что за таблица (лист, колонки) · **(2)** что
сделать · **(3)** что на выходе (новый лист / покраска / колонка) · **(4)** особые
правила. Плохой вход: «сделай макрос для проводок». Хороший: «на листе Данные,
колонка D — суммы; выдели жёлтым выше 100000; исходные не трогай».

### Практика (по желанию/по времени)
1. Желающие описывают агенту свою микро-задачу по формуле входа.
2. Разбор 1–2 ответов: достаточно ли данных подали, что переспросил агент.
   (Вставку кода в VBA массово в зале НЕ гоняем — это уносят на потом.)

### Безопасность (проговорить)
Агенту — только обезличенные данные или общее описание, не реальные таблицы с УНП.
