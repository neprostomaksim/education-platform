## М2. Договоры → таблица условий (75 мин) — ЯДРО ДНЯ

### Подцель
Аудитор собирает агента-экстрактор под свои таблицы и прогоняет договор в строку
рабочего документа. Ключевая идея: **список колонок таблицы = ТЗ для агента.**

### LMS-опора
- **Карточки-эталоны:** колонки обеих таблиц (кредитные договоры — 16 полей;
  договоры с контрагентами — 13 полей).
- **Правило аудиторского агента:** не выдумывать · существенное дословно · ставить
  пункт-источник · нет данных → «не указано» · противоречие → «проверить вручную».
- **Два готовых промпта:** экстрактор кредитных договоров и экстрактор договоров
  с контрагентами (файл `Промпты_агентов_Gem`, №1 и №2).
- **Учебный договор** (обезличенный) для демо и практики.

### Демо (последовательность шагов)
1. Показать эталонную таблицу условий кредитных договоров — обратить внимание, что
   **колонки = поля**, которые надо извлечь.
2. Создать Gem «Экстрактор кредитных договоров»: Gems Manager → New Gem → вставить
   промпт №1 → сохранить.
3. Подать в агент **обезличенный учебный договор**.
4. Разобрать ответ по трём частям вывода:
   - таблица «поле → значение [пункт]»;
   - строка через табуляцию — показать вставку в Excel (колонки B:Q);
   - блок «⚠️ Требует проверки аудитором».
5. **Момент доверия** (главный): показать, что по незаполненным полям (график
   погашения, цель) агент написал «не указано», а по ставке — «проверить вручную»
   с двумя местами (п.4.1 vs доп.соглашение). Агент не выдумал.

### Практика (шаги)
1. Каждый создаёт свой Gem-экстрактор (кредитный или контрагентский — по выбору).
2. Прогоняет учебный (или свой обезличенный) договор.
3. Вставляет строку-результат в таблицу условий.
4. **Чиним типовое:** агент «разговорчивый» (добавляет вступления) → в промпте
   усилить «ФОРМАТ ВЫВОДА без вступлений»; агент пересказал вместо цитаты → напомнить
   правило «существенное дословно».

### Сквозной принцип (проговорить)
**Список колонок = ТЗ для агента.** Понял это — соберёшь экстрактор под ЛЮБУЮ таблицу:
меняешь список полей в промпте — получаешь новый агент. Это переносится на реестры,
описи, любые «извлечь из документа в таблицу».

### Материалы модуля
Эталонные таблицы (колонки) · промпты №1 и №2 · учебный договор · правило
аудиторского агента.

## Агент «Формулы и сводные» (15 мин)

### Подцель
Естественное продолжение М2: таблицу собрали — теперь быстро посчитать по ней без
кода. Аудитор описывает задачу → агент выбирает простейший путь (формула или сводная)
и даёт готовую формулу / пошаговые клики. Промпт — `Промпты_агентов_Gem`, №12 (Windows).

> Пара к генератору макросов из занятия 1: тот пишет код, когда без него никак; этот
> решает **без кода**, когда можно проще. Проговорить это различие.

### Демо (последовательность шагов)
1. Взять таблицу (например, результат М2 или любую с суммами по контрагентам).
2. Создать Gem «Формулы и сводные» (промпт №12).
3. Задача-формула: «на листе Данные: B — контрагент, D — сумма. Хочу общую сумму по
   каждому контрагенту» → показать, что агент даёт **готовую формулу** (СУММЕСЛИ)
   или предлагает **сводную** с пошаговыми кликами.
4. Подчеркнуть: агент НЕ пишет макрос, где хватает формулы — выбирает простейшее.

### Формула входа
**(1)** что за таблица (лист, колонки) · **(2)** что посчитать/найти · **(3)** если
таблиц несколько — назвать оба листа. Плохой вход: «посчитай по контрагентам».
Хороший: «лист Данные, B — контрагент, D — сумма, хочу сумму по каждому».

### Практика (по времени)
Каждый задаёт агенту свою задачу по формуле входа, вставляет полученную формулу /
собирает сводную по шагам. Разбор: где формула, а где уместнее сводная.

## NotebookLM — работа с документами (25 мин)

### Подцель
Освоить «спроси у своих документов»: быстро разбираться в объёмных документах и НПА
без галлюцинаций. **Переносимый навык.**

### LMS-опора
- **Что это:** бесплатный инструмент, отвечает ТОЛЬКО по загруженным источникам и
  даёт ссылку на фрагмент. Нет ответа в источнике — говорит «не найдено».
- **Настройка чата → Свой вариант:** блокноту можно задать роль и правила (как
  системный промпт, до 10 000 символов) — тогда он отвечает дисциплинированно и
  только по загруженным документам. Готовые инструкции — файл `NotebookLM_агенты`
  (НПА, договоры, учётная политика, сравнение документов).
- **Честные границы:** NotebookLM — не живая правовая база. Действует ли акт «сегодня»
  — он не знает; актуальную редакцию берёшь в Эталоне, а NotebookLM помогает быстро
  в ней разобраться и применить.
- **Памятка:** как создать блокнот, добавить источник (PDF/копипаст/ссылка), задавать
  вопросы.

### Демо (последовательность шагов)
1. New Notebook → Add source → загрузить **конкретный НПА** (твой пример) + при
   желании методичку фирмы.
2. **Настройка чата → Свой вариант** → вставить инструкцию агента по НПА (из файла
   `NotebookLM_агенты`) → сохранить. Показать, что блокнот теперь держит роль и правила.
3. Задать вопросы:
   - «Что дословно сказано в пункте ___? Дай цитату со ссылкой.»
   - «Какая ответственность/порядок предусмотрены по этому документу?»
   - «Есть ли переходные положения или дата вступления в силу?»
4. **Провокация (главный момент):** спросить то, чего в документе НЕТ → показать ответ
   «в источниках не найдено». Вот это и есть grounding — доверие к инструменту.
5. Мораль: цитату из ответа → в замечание (через агента-редактора, Часть 3);
   актуальность редакции → подтверждаешь в правовой базе.

### Практика (шаги)
1. Каждый создаёт блокнот и грузит свой документ (регламент, положение, НПА).
2. Задаёт 2 вопроса, включая один «на слабо» — чего в документе точно нет.
3. Убеждается, что инструмент не выдумывает.

### Переносимость (проговорить)
Тот же приём — на любые документы: учётная политика, договоры, длинные PDF,
внутренние регламенты. Уносят в любую задачу.

### Материалы модуля
Памятка NotebookLM · инструкции агентов (`NotebookLM_агенты`: НПА, договоры,
учётная политика, сравнение) · честные границы · пример НПА для загрузки (твой).
