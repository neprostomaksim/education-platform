# Макросы: как пользоваться и как они устроены

> Два уровня. **Часть 1–2** — как запустить и как читать любой макрос (общие
> «кирпичики»). **Часть 3** — разбор каждого макроса: что делает, как работает
> по шагам, что и где менять. Код всех макросов — в файле `Макросы_весь_код`.

## Оглавление
- Часть 1. Как запустить любой макрос
- Часть 2. Общие «кирпичики» (чтобы читать код)
- Часть 3. Разбор каждого макроса
  - 3.1 Анонимизация
  - 3.2 JET — тесты проводок
  - 3.3 Сверка ОСВ
  - 3.4 Сверка двух списков
  - 3.5 Аудиторская выборка

## Часть 1. Как запустить любой макрос

1. Открыть нужный Excel-файл.
2. Нажать `Alt+F11` — откроется редактор макросов (VBA).
3. Меню Insert → Module — создастся пустой модуль.
4. Скопировать код нужного макроса из `Макросы_весь_код` и вставить в модуль.
5. Закрыть редактор (крестик), вернуться в Excel.
6. Нажать `Alt+F8` — список макросов → выбрать нужный → **Выполнить (Run)**.

**Если Excel блокирует макросы:** Файл → Параметры → Центр управления безопасностью →
Параметры макросов → «Отключить с уведомлением» (тогда при открытии будет кнопка
«Включить содержимое»). В некоторых корп-средах это закрыто политикой — тогда только
на своей машине или запасным ручным путём.

> **Важно про имена листов.** Каждый макрос ищет данные на листе с конкретным именем
> (например, JET — на листе «Проводки», ОСВ — на «ОСВ»). Если лист назван иначе —
> макрос не найдёт данные. Либо переименуй лист, либо поменяй имя в коде (см. Часть 2).

## Часть 2. Общие «кирпичики» (одинаковые во всех макросах)

Понимаешь эти 7 штук — читаешь любой наш макрос.

**1. Переменные — `Dim`.**
`Dim r As Long` — «заведи ящик по имени r для целого числа». `As String` — для текста,
`As Double` — для чисел с копейками, `As Worksheet` — для листа. Это просто именованные
коробки, куда кладём значения.

**2. Ячейки — `Cells(строка, колонка).Value`.**
`Cells(5, 3).Value` — значение ячейки на 5-й строке, 3-й колонке (это колонка C).
Колонки нумеруются числами: A=1, B=2, C=3, D=4… Прочитать = взять `.Value`,
записать = присвоить `.Value = "текст"`.

**3. Цикл — `For … Next` / `For Each … Next`.**
«Пройди по всем строкам/ячейкам и сделай одно и то же». `For r = 2 To 100` — от строки 2
до 100. `For Each cell In Selection` — по каждой выделенной ячейке. Внутри — что делаем
с каждой.

**4. Условие — `If … Then … End If`.**
«Если выполняется — сделай». `If Сумма > порог Then ЗаписатьФлаг…` — если сумма больше
порога, пометь. `Else` — иначе.

**5. Словарь — `Dictionary`.**
Справочник «ключ → значение», как телефонная книга. Используем, чтобы:
считать повторы (ключ = значение, значение = счётчик), убирать дубли, хранить
пары (токен → оригинал в анонимизации; документ → сумма в сверке). Быстрый способ
«а видели мы уже такое?».

**6. Окна — `MsgBox` / `InputBox`.**
`MsgBox "текст"` — показать сообщение (итог работы). `InputBox "вопрос"` — спросить у
пользователя значение (префикс токена, порог, метод выборки).

**7. Лист результатов + помощник `ПересоздатьЛист`.**
Почти каждый макрос пишет результат на отдельный лист («Флаги», «Результат_сверки»…).
Помощник `ПересоздатьЛист("имя")` удаляет старый такой лист (чтобы не мешался прошлый
прогон) и создаёт чистый. Это служебная функция, логику теста не трогает.

> В коде ещё встречаются `Trim` (убрать пробелы по краям), `IsNumeric` (это число?),
> `Abs` (модуль числа), `Format` (привести к виду, напр. «000» → 001). Это мелкие
> помощники, по названию понятно, что делают.

## Часть 3. Разбор каждого макроса

### 3.1 Анонимизация

**Что делает.** Заменяет реальные значения на токены (КОМПАНИЯ_001) перед загрузкой в
нейросеть и умеет вернуть обратно.

**Как запустить.**
- `Обезличить`: выделить диапазон → запустить → ввести префикс (КОМПАНИЯ, ФИО, СЧЁТ).
- `ВернутьДанные`: выделить ячейки с токенами → запустить.

**Как работает (по шагам).**
1. Берёт выделенный диапазон.
2. Спрашивает префикс токена.
3. Идёт по каждой ячейке. Через словарь проверяет: это значение уже встречалось?
   - если нет — присваивает новый токен (префикс + номер 001, 002…), запоминает пару
     на листе «Карта_замен»;
   - если да — ставит тот же токен, что и в прошлый раз (одинаковое → одинаковый токен).
4. `ВернутьДанные` читает «Карта_замен» в словарь и в каждой ячейке заменяет токены
   обратно на оригиналы (даже если токен сидит внутри текста).

**Что можно менять и где.**
- **Формат номера токена:** `Format(counter, "000")` → «000» даёт 001. Хочешь 4 знака —
  поставь «0000».
- **Что маскировать:** это не в коде, а в том, какой диапазон ты выделяешь. Выделяй
  колонки с чувствительными данными.
- **Ограничение:** маскирует значение ячейки целиком. Если контрагент/УНП внутри длинной
  строки — не поймает. Это осознанное ограничение, проверяем глазами.

### 3.2 JET — тесты проводок

**Что делает.** Прогоняет 11 механических тестов + Бенфорд по выгрузке проводок 1С,
собирает срабатывания на лист «Флаги».

**Как запустить.** `ПрогнатьВсеТесты` (всё разом) или отдельный `ТестNN` под кнопку.

**Как работает (по шагам).**
1. **`ПодготовитьДанные` — самый важный шаг.** Выгрузка 1С «грязная»: одна проводка
   занимает несколько строк (первая — дата/сумма/счета, следующие — договор, ТТН,
   ставка). Макрос «схлопывает» это в одну строку на проводку на чистом листе
   «JET_данные»: находит шапку (ячейку «Дата»), собирает главные строки, а доп. строки
   подклеивает в поле «Первичка». Дату разбирает из текста «03.01.2025 8:40:25».
2. **Каждый тест** идёт по чистой таблице и применяет своё правило:
   - выходной — день недели суббота/воскресенье или дата в списке праздников;
   - круглая/999 — сумма кратна 1000 или заканчивается на 999;
   - крупная — больше порога существенности;
   - повтор — одинаковые «содержание+сумма» встречаются >1 раза (через словарь-счётчик);
   - Дт=Кт — совпадают счета дебета и кредита;
   - редкая пара — корреспонденция Дт→Кт встречается ≤ порога (через словарь-счётчик);
   - сторно — отрицательная сумма; без описания — пустое «Содержание»; без суммы — пусто;
   - полнота — нет Дт/Кт/даты.
3. **Бенфорд** считает первые цифры всех сумм и сравнивает с ожидаемым распределением;
   пишет таблицу на «JET_Бенфорд», подсвечивает отклонения > 8 п.п.
4. Все срабатывания пишутся на «Флаги» (строка = одно срабатывание: дата, документ,
   сумма, тест, комментарий).

**Что можно менять и где.**
- **Порог существенности и список праздников:** на листе `JET_настройки` (создаётся
  сам). B1 — порог (руб.), B2 — порог «редкой» пары, с 5-й строки — праздники (дд.мм).
  Меняешь прямо в Excel, код трогать не надо.
- **Чувствительность теста «крупные»:** это и есть порог из B1.
- **Порог 999 / круглых:** в коде теста `Тест04` — `Mod 1000` (кратность тысяче). Хочешь
  ловить кратные 100 — поменяй на `Mod 100`.
- **Если у другого клиента другой порядок колонок:** вверху кода константы `cДата`,
  `cДт`, `cСумма` и т.п. + в `ПодготовитьДанные` номера колонок при чтении из «Проводки».
  Это единственное место, где макрос завязан на их формат выгрузки.
- **Формат даты:** функция `ПарсДату` ждёт «дд.мм.гггг». Другой формат — поправить там.

### 3.3 Сверка ОСВ

**Что делает.** Проверяет арифметику оборотно-сальдовой ведомости: где строка не
сходится и бьётся ли баланс.

**Как запустить.** `ОСВ_Проверка` (данные на листе «ОСВ»).

**Как работает (по шагам).**
1. Находит шапку (ячейку «Код»).
2. По каждой строке считает «нетто» = Дт минус Кт для трёх пар колонок: начало (D−E),
   оборот (F−G), конец (H−I).
3. **Главная проверка — разбаланс:** конец должен равняться начало + оборот. Если
   расхождение больше копейки — флаг.
4. Ещё флажит: пустые строки (всё по нулям), развёрнутое сальдо (и Дт, и Кт на конец).
5. **Контроль баланса:** суммирует нетто-конец только по синтетическим счетам (код без
   точки — 01, 62, а не 01.1). У правильной ОСВ эта сумма = 0. Если нет — «баланс не
   сходится».
6. Пишет всё на «ОСВ_флаги».

**Почему только синтетические в балансе.** Субсчета (01.1, 01.2) дублируют суммы своего
счёта (01). Если сложить всё подряд — получишь тройной счёт и ложное расхождение.
Поэтому баланс — только по счетам верхнего уровня.

**Что можно менять и где.**
- **Точность (допуск):** константа `ДОПУСК = 0.01` вверху. Хочешь считать расхождением
  только от рубля — поставь `1`.
- **Если колонки ОСВ в другом порядке:** в коде номера колонок в функции `Netto`
  (4,5 — начало; 6,7 — оборот; 8,9 — конец).
- **Убрал «транзит» намеренно** — он давал бы сотни ложных флагов (счета доходов/расходов
  закрываются в ноль по природе). Если захочешь вернуть — это отдельное правило.

### 3.4 Сверка двух списков

**Что делает.** Сопоставляет наш учёт и данные контрагента, показывает расхождения.

**Как запустить.** `Сверка_Списков` (листы «Наш_учет» и «Контрагент», формат:
A Контрагент | B Документ-ключ | C Сумма).

**Как работает (по шагам).**
1. Загружает оба листа в словари: ключ = номер документа (колонка B), значение =
   контрагент + сумма. Если ключ повторяется — суммы складываются.
2. Объединяет все ключи из обоих списков.
3. По каждому ключу смотрит:
   - есть в обоих: суммы совпали (в пределах допуска) → тихо; разошлись → флаг с обеими
     суммами и разницей;
   - только у нас → «есть у нас, нет у контрагента»;
   - только у них → «есть у контрагента, нет у нас».
4. Пишет проблемы на «Результат_сверки», итог — в окне.

**Что можно менять и где.**
- **Допуск расхождения:** константа `ДОПУСК = 0.01`. Часто ставят 1 руб. — тогда
  копеечные округления не всплывают.
- **Ключ сопоставления:** сейчас колонка B (документ). Если сверяешь по другому полю —
  поменять номер колонки в функции `ЗагрузитьСписок`.
- **Имена листов:** «Наш_учет» и «Контрагент» — если у тебя иначе, поправить в коде.

### 3.5 Аудиторская выборка

**Что делает.** Отбирает элементы из совокупности тремя методами.

**Как запустить.** `Выборка` (лист «Совокупность», сумма в колонке D) → спросит метод.

**Как работает (по методам).**
1. **Метод 1 — сплошь выше порога.** Берёт все позиции, где сумма ≥ порога
   (существенные проверяем сплошняком).
2. **Метод 2 — случайная N.** Из позиций НИЖЕ порога перемешивает и берёт N штук
   (перемешивание — алгоритм Фишера-Йейтса, честный рандом).
3. **Метод 3 — монетарная (MUS).** Считает шаг = общая сумма / N, ставит случайную
   стартовую точку и «шагает» по накопленной сумме, отбирая позицию на каждом шаге.
   Чем крупнее сумма — тем вероятнее попадёт (отбор пропорционален деньгам).
4. Пишет отобранное на «Выборка_результат» с колонкой «Метод отбора».

**Что можно менять и где.**
- **Колонка суммы:** константа `КОЛ_СУММА = 4` (колонка D). Другая колонка — поменять
  число.
- **Порог и N:** макрос спрашивает при запуске, в коде фиксировать не нужно.
- **Комбинированный подход:** на практике часто «сплошь выше порога + случайная из
  остатка». Сейчас это два прогона (метод 1, потом метод 2) — результаты можно склеить.

## Короткая шпаргалка «что где крутить»

| Хочу поменять | Где |
|---|---|
| Порог существенности JET, праздники | лист `JET_настройки` (в Excel) |
| Порог/кратность круглых сумм | код `Тест04`, `Mod 1000` |
| Порядок колонок под другого клиента (JET) | константы `cДата…` + `ПодготовитьДанные` |
| Точность сверки ОСВ / списков | константа `ДОПУСК` в коде |
| Ключ сверки списков | колонка в `ЗагрузитьСписок` |
| Колонка суммы в выборке | константа `КОЛ_СУММА` |
| Формат номера токена | `Format(counter, "000")` в анонимизации |
| Имена листов | вверху каждого макроса (константы `SH_…`) |

> Общий принцип правок: **сначала пробуй менять то, что вынесено в настройки-листы или в
> константы вверху кода** — это безопасно. Логику внутри тестов трогай, только когда
> понял, что делает конкретный кусок.
